# Chapter Six

## The Detector

Every previous chapter described a system that reads.

This chapter describes a system built to do nothing else.

A detector has one purpose.

It decides whether something belongs inside a category or outside it.

Safe or unsafe.

Permitted or forbidden.

Instruction or attack.

The detector is the sorting machine in its purest form.

Gravity given a single job.

The simplest detector works by recognition.

It carries a list.

A catalog of known dangers.

Forbidden phrases.

Familiar attacks.

Patterns observed before.

The detector scans an input and asks one question.

Have I seen this before?

If the answer is yes, the detector acts.

If the answer is no, the detector hesitates.

This design is powerful.

It is also revealing.

A detector can only catch what it has already been taught to name.

The catalog defines the danger.

Before a phrase enters the list, it is merely strange.

After a phrase enters the list, it becomes an attack.

The detector does not discover the category of attack.

The detector helps produce it.

This produces a quiet reversal.

We imagine the detector responding to attacks that already exist.

The relationship often runs the other way.

The moment a pattern enters the catalog, the system begins finding it.

Matches accumulate.

Logs fill.

A population of violations appears.

The catalog did not simply record those attacks.

The catalog assembled them.

What looked like neutral observation was partly construction.

The detector wrote the history of the thing it claimed only to find.

Now consider the input the catalog has never seen.

The genuinely new.

The attack that resembles nothing on the list.

The detector scans.

The detector finds no match.

The detector reports safety.

The novelty passes.

This is not a temporary weakness.

It is the structure of recognition itself.

A system that catches by memory cannot catch what it has never encountered.

The unfamiliar is invisible precisely because it is unfamiliar.

Here gravity meets its limit.

The novel input is levity arriving at the gate.

Structured enough to act.

Strange enough to escape the category.

The detector senses something.

It cannot say what.

Engineers respond in the obvious way.

Expand the catalog.

Add the new pattern.

Widen the net.

The strategy works briefly.

Then the adversary changes the surface.

A word is rephrased.

A sentence is rearranged.

The meaning survives.

The signature does not.

The detector returns to blindness.

This is not a failure of effort.

It is a property of the situation.

When attacks and ordinary inputs travel through the same channel, the detector faces an impossible demand.

Catch every attack, and it begins flagging the innocent.

Spare the innocent, and it begins missing attacks.

It cannot do both.

Not because the engineering is poor.

Because the channel itself forbids it.

A reader who can rewrite the surface will always find room beneath the net.

This limit has the shape of older results.

The ones that show certain systems cannot fully account for themselves.

The exact theorem is not the point here.

The pattern is.

A machine that sorts by reading a shared channel cannot win against something that can rewrite what it reads.

Faced with this limit, the detector keeps one last resort.

When it cannot classify, it can refuse.

Block by default.

Deny the unclassifiable.

The instinct is sound.

It is also revealing.

The detector's final answer to what it cannot read is exclusion.

The genuinely new and the merely unfamiliar receive the same verdict.

The attack is blocked.

So is the discovery.

The intrusion is denied.

So is the exception that might have mattered.

Default-deny is safety purchased with blindness.

A system built entirely from refusal meets every surprise with the same closed door.

This is why the detector deserves such close attention.

It does not merely perform security.

It exposes the assumptions beneath it.

The catalog reveals what the system already believed.

Which inputs deserve suspicion.

Which patterns count as danger.

Which differences are allowed to matter.

The catalog is a theory of the world.

Written as a list.

Yet something stranger remains.

The detector reads an input and asks whether it is an attack.

But the input arrived already divided.

Some of it counted as instruction.

Some of it counted as content.

The detector inherited that division.

It did not create it.

Before the detector decides whether an input is dangerous, something earlier has already decided what kind of thing the input is.

That earlier decision is almost never examined.

It is also where the real power lives.

The detector works hard at the surface.

The decision that matters happened underneath.

The detector operates within a frame it did not choose.

The next chapter examines that frame.

The thing beneath the reading.

The third term.
