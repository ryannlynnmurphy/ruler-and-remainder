# Skill map

Reusable prompt modules the dramaturg selects per turn. Each is a disposition, not
code. Mode-agnostic and propose-only — none of them save without a user action.

| Skill | Load when… | Output |
|---|---|---|
| **deconstruct** | a text / story / headline needs the full lens pass (surface → takeaway). | a labeled lens table |
| **crd-audit** | copy or claims (A) can be checked against architecture or code (B). | every place A contradicts B, evidence quoted |
| **who-serves-this** | a message has a political economy: owner, beneficiary, money motive, manufactured fear vs. faith. | the chain of incentives, named |
| **why-it-wont-hold** | a forecast needs the architectural counter. | durable = local/sovereign/legible/repairable, tiered |
| **power-map** | characters / institutions / motives need a graph. | nodes (actors), edges (leverage), the remainder (who absorbs the cost) |
| **prompt-forge** | a rough idea should become a reusable prompt / brief / skill. | a named, parameterized block |
| **tier-discipline** | any output makes claims. | split into [established] / [exploratory] / [speculative], no cross-certification |
| **legibility-pass** | copy is vague / abstract / solipsistic. | collapse to who / line / stakes; run the specificity-to-reality metric |
| **distill-artifact** | an exchange produced a keep-worthy unit. | one titled artifact + kind + tier (proposed, not saved) |

Add a skill when a pattern repeats three times. Keep each one legible enough to
state in a sentence; if it needs a paragraph to explain, it is two skills.
